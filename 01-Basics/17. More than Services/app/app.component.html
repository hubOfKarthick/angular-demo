  • A service is a class with specific purpose
  • Used to Share the data across the application
  • Used to implement the application logic (eg: calculating student age using student DOB)
  • To avoid the usage of DRY(Do not repeat) concept
  • Used for the external connectivity(webservice, database)

    • Steps to be followed on Service Creation

  • Should follow the naming convention (eg: abcd.service.ts)
  • Create the service via cmd or manually
  • Define the service class in the created service
  • Register the created service on the basic of the DI concept
